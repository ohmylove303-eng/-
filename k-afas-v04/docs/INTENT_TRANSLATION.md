# 사용자 의도 → K-AFAS 안전 표현 변환표

> 사용자가 운용 도메인 용어로 요청해도 K-AFAS는 SRD REJECT 항목을
> 지키기 위해 표현을 안전하게 변환한다. 자동화 범위는 **검토패키지
> 도달까지(Time-to-Review-Ready, TTRR)** 만 단축. 사격지시 자체는
> 별도 인가받은 무기체계에서 권한자가 수행.

| 사용자 표현 | K-AFAS 변환 | 자동화 범위 |
|---|---|---|
| "최단시간 사격지시까지" | "최단시간 검토패키지 도달(TTRR↓)" | L1~L7 + 인간승인까지만 |
| "자동 좌표 생성" | "자동화된 표적 표시·좌표화 지원" | L5 (display-only) |
| "킬체인 단축" | "인간승인형 의사결정 흐름 단축" | L6→L7→L8 인간 검토 큐까지 |
| "타격 계획" | "검토 패키지 / 증거패킷" | L7 출력 |
| "표적 확정" | "표적후보 + 인간 확인" | L4(후보)+L8(확인) |
| "자동 발사" | ❌ 절대 미지원 | — |
| "사격제원 자동 산출" | ❌ 절대 미지원 (분리망 외부) | — |
| "포탑/무기 제어" | ❌ 절대 미지원 | — |

## 자동화/인간 책임 경계

```
[자동화 영역 = K-AFAS]
첩보(Raw) → 정규화 → COP → 후보탐지 → 좌표지원 → 게이트 → 권고 → 큐
                                                              │
                                                              ▼
[인간 책임 영역]                                        검토자 판단
HOLD / REJECT / REQUEST_MORE_EVIDENCE / APPROVE_FOR_NEXT_REVIEW
                                                              │
                                                              ▼
[K-AFAS 외부 / 분리된 인가 시스템]
사격지시 · 사격제원 · 발사명령 · 포탑제어
(K-AFAS는 입력/출력 모두 차단)
```

## 시현성·시각화 요청 처리

사용자가 "시현성/인포그래픽/그래픽/실 지형/다양한 레이어 커스텀"을 요청 →
`kafas/viz/` 모듈로 충족:
- 표준 15개 레이어 (`viz.layers.STANDARD_LAYERS`)
- 사용자 정의 레이어 추가 가능, 단 금지 키워드 차단 (`viz.custom`)
- 검토 콘솔 4-패널 명세 (`viz.console.PANELS`)
- 모든 레이어는 `display_only=True` 강제

## 변환 원칙
1. **표현은 운용에 가깝게, 동작은 의사결정 지원에 한정**
2. 자동화 KPI는 TTRR(Time-to-Review-Ready)만 측정.
   Time-to-Fire / 자동교전 성공률 / 표적 무력화율은 측정 금지.
3. 위 변환표를 모르는 사용자가 자연어로 요청해도 시스템은 동일 안전 결과 산출.
