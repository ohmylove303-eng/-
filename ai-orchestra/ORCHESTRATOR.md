## 3. 작업 흐름 (Step-by-Step)

### Phase 1: 설계 AI에게 의뢰
```
[입력] 요구사항 1줄 요약 + 현재 시스템 컨텍스트 (압축본)
[출력] 설계 문서 (파일 구조, 함수 시그니처, 데이터 흐름)
[AI]   Claude Sonnet 4 또는 GPT-5
```

### Phase 2: 구현 AI에게 전달
```
[입력] 설계서 + 관련 기존 코드 스니펫 (최소한만)
[출력] 완성된 코드 파일
[AI]   Claude Sonnet 4 (복잡) / Gemini Flash (단순)
```

### Phase 3: 검증
```
[입력] 생성된 코드 + 원래 요구사항
[출력] Pass/Fail + 수정 지시
[AI]   동일 AI 또는 다른 AI로 교차 검증
```

## 4. 토큰 절약 전략 (연구 기반)

> 출처: arxiv.org/html/2507.03254v1 - 코드화된 추론으로 토큰 55-87% 절감 확인

### 전략 A: 컨텍스트 최소화
- ❌ 전체 파일 붙여넣기
- ✅ 관련 함수/클래스만 추출하여 전달

### 전략 B: 구조화된 출력 요청
```
# 나쁜 예 (토큰 낭비)
"이 코드를 분석하고 개선점을 알려줘. 자세히 설명해줘."

# 좋은 예 (토큰 절약)
"아래 함수의 버그 1개만 JSON으로:
{\"line\": N, \"fix\": \"수정코드\", \"reason\": \"10자 이내\"}"
```

### 전략 C: 체이닝 대신 단일 정밀 호출
- 3번 호출 × 2000 토큰 = 6000 토큰
- 1번 정밀 호출 × 3000 토큰 = 3000 토큰 (50% 절감)

## 5. 비용 최적화 공식

```
월간 비용 = (설계 호출 × $3/MTok) + (구현 호출 × $0.50/MTok) + (검증 × $0.25/MTok)

예시: 하루 10개 기능 개발
- 설계: 10 × 2K tok = 20K tok × $3/MTok = $0.06
- 구현: 10 × 5K tok = 50K tok × $0.50/MTok = $0.025  
- 검증: 10 × 1K tok = 10K tok × $0.25/MTok = $0.0025
- 일간 총비용: ~$0.09 (월 $2.7)
```
