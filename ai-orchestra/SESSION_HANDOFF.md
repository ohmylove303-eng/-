# 세션 핸드오프 시스템 (30주제 전환 규칙)

## 왜 필요한가?
- AI 컨텍스트 윈도우는 유한함 (128K~200K 토큰)
- 30개 이상 주제 시 초기 맥락 손실 → 품질 저하
- 새 세션으로 전환하되, 핵심 맥락을 100% 이어받는 시스템

---

## 1. 전환 타이밍 감지

| 신호 | 행동 |
|------|------|
| 주제 25개 도달 | ⚠️ 경고: "5개 주제 후 세션 전환 예정" |
| 주제 30개 도달 | 🔄 핸드오프 문서 생성 → 새 세션 시작 |
| AI 응답 품질 저하 감지 | 🔄 즉시 전환 (반복, 환각, 맥락 잊음) |
| 완전히 다른 주제 시작 | 🔄 선제적 전환 권장 |

---

## 2. 핸드오프 문서 생성 프롬프트

### 현재 세션 종료 시, AI에게 이렇게 요청:

```markdown
세션을 종료한다. 다음 세션을 위한 핸드오프 문서를 만들어줘.

포함할 내용:
1. 프로젝트 컨텍스트 (L2 압축)
2. 이번 세션 완료 작업 리스트
3. 미해결 이슈
4. 핵심 결정사항 (번복 금지 사항들)
5. 다음 세션 첫 작업
6. 수정된 파일 목록

출력 형식: 마크다운, 500토큰 이내
```

---

## 3. 핸드오프 문서 템플릿

```markdown
# SESSION HANDOFF #N → #(N+1)
날짜: YYYY-MM-DD

## 프로젝트
US Stock Dashboard / Flask+Python / Gemini AI
GitHub: ohmylove303-eng/-

## 완료
1. ✅ telegram_alert.py 생성 (scripts/)
2. ✅ update_all.py에 스크립트 등록
3. ✅ CI/CD에 alert step 추가
4. ✅ 매크로 분석 한/영 전환 완료

## 미해결
1. ❗ VCP 스크리너 timeout 간헐적 발생 (line 45, yfinance)
2. ❗ 프론트엔드 다크모드 미구현

## 핵심 결정 (번복 금지)
- 알림 threshold: 5%
- AI 모델: Gemini 2.0 Flash (메인), OpenAI (폴백)
- 데이터 갱신 주기: 6시간
- 배포: Render + GitHub Actions

## 다음 세션 첫 작업
"프론트엔드 다크모드 구현. templates/index.html + static/js/dashboard.js 수정"

## 수정 파일
- scripts/telegram_alert.py (신규)
- scripts/update_all.py (수정: line 18에 추가)
- .github/workflows/daily_update.yml (수정: alert step)
```

## 4. 새 세션 시작 프롬프트 (복사해서 사용)

```markdown
# 이전 세션 이어서 작업

## 시스템 역할
너는 US Stock Dashboard 프로젝트의 시니어 개발자다.
아래 핸드오프 문서를 읽고 이전 세션을 이어서 작업한다.

## 핸드오프 문서
{여기에 핸드오프 문서 붙여넣기}

## 규칙
1. 핸드오프의 "핵심 결정"은 번복하지 마라
2. "다음 세션 첫 작업"부터 시작하라
3. 미해결 이슈를 인지하고, 관련 작업 시 해결을 시도하라
4. 설계가 필요하면 DESIGN_AI_PROMPT.md 형식을 따르라
5. 구현이 필요하면 IMPLEMENT_AI_PROMPT.md 형식을 따르라

## 첫 작업 시작
{구체적 지시}
```

## 5. 주제 카운팅 방법

```
주제 = 사용자의 새로운 요청 1개
예외: 같은 요청에 대한 후속 질문/수정은 같은 주제

카운팅 예시:
#1 "telegram 알림 추가해줘" ← 새 주제
#1 "에러 발생했어, 수정해줘" ← 같은 주제 (후속)
#2 "매크로 분석에 영어 지원 추가" ← 새 주제
#3 "다크모드 구현해줘" ← 새 주제
```

## 6. 하네스(Harness) 기법 적용

### 하네스 = 출력 형식을 강제하여 품질 유지

```markdown
# 모든 응답에 적용할 하네스:

## 응답 시작 전 체크리스트 (내부 사고)
- [ ] 현재 주제 번호: #N
- [ ] 컨텍스트 충분한가?
- [ ] 설계가 필요한가 vs 바로 구현 가능한가?
- [ ] 어떤 AI가 최적인가?

## 응답 형식 강제
1. 판단: [설계필요/바로구현/정보요청]
2. 사용할 AI: [모델명]
3. 프롬프트: [적용할 템플릿]
4. 출력물: [코드/설계서/답변]
5. 검증: [동작확인방법]
```
